Documentation for DC/OS Minio 0.1.0

DC/OS Minio Service is an automated service that makes it easy to deploy and manage Minio on Mesosphere DC/OS. For more information on Minio, see the Minio documentation.

WARNING: This is a Community service. Community services are not tested for production environments. There may be bugs, incomplete features, incorrect documentation, or other discrepancies.

Benefits

DC/OS Minio offers the following benefits:

  1. Based on the proven DC/OS framework/operator SDK underpinning all DC/OS-certified frameworks
  2. Minio is a lightweight object storage server
  3. Minio deploys Amazon S3-compatible object storage servers in seconds
  4. Provides consistent deployment across multi-cloud environments
  5. Backup and restore operations use AWS S3 as backing store

DC/OS Minio’s main features are:

  1. Erasure Code: DC/OS Minio protects data against hardware failures and silent data corruption using erasure code and checksums.
  2. Bit rot protection: DC/OS Minio provides bit rot protection using the HighwayHash algorithm.
  3. Bucket Notification Service: DC/OS Minio provides a bucket notification service. Events occurring on objects in a bucket can be monitored using bucket event notifications.
  4. Gateway: DC/OS Minio adds Amazon S3 compatibility to Azure Blob Storage, GCP and NAS.
  5. Large Bucket Support: Allows a single DC/OS Minio bucket to expand over multiple erasure code deployment sets.
  6. Disk Cache: Disk caching allows caching disks to store content closer to tenants.
  7. Monitoring: DC/OS Minio server exposes monitoring data over unauthenticated endpoints, so monitoring tools can pick the data without you having to share DC/OS Minio server credentials.

Dynamic Scaling Out

DC/OS Minio does not support horizontal scaling; if a DC/OS Minio cluster has started with eight DC/OS Minio servers then a ninth DC/OS Minio server cannot be added to the cluster.
